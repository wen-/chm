centos6.5系统php模块扩展

通过yum安装了LAMP环境后要扩展PHP的模块（如：curl、mcrypt等），操作如下：

以安装mcrypt扩展为例：

软件包

php-5.3.3.tar.gz（对应yum安装的版本）

libmcrypt(libmcrypt-2.5.8.tar.gz )：

mhash(mhash-0.9.9.9.tar.gz ):
mcrypt(mcrypt-2.6.8.tar.gz ):

一、源码编译安装PHP对应（yum）的版本，保留解压后的包。安装路径为：/usr/local/php/

二、按顺序安装

1 .先安装Libmcrypt

#tar -zxvf libmcrypt-2.5.8.tar.gz
#cd libmcrypt-2.5.8
#./configure
#make
#make install 说明：libmcript默认安装在/usr/local 
 
缺少文件时请安装：[root@localhost libmcrypt-2.5.8]# yum -y install gcc gcc-c++ 


2.安装mhash

#tar -zxvf mhash-0.9.9.9.tar.gz
#cd mhash-0.9.9.9
#./configure
#make
#make install

3.安装mcrypt

#tar -zxvf mcrypt-2.6.8.tar.gz
#cd mcrypt-2.6.8
#LD_LIBRARY_PATH=/usr/local/lib ./configure
#make
#make install

 

三、进入PHP解压出来的mcrypt文件夹

[root@localhost ~]# cd php-5.3.3/ext/mcrypt/

执行phpize命令（phpize是用来扩展php扩展模块的，通过phpize可以建立php的外挂模块，如果没有？在执行第一步编译安装PHP时已生成【/usr/local/php/bin/phpize】）

[root@localhost mcrypt]# whereis phpize    //为了确定phpize存在,如果不存在去【/usr/local/php/bin/phpize】
phpize: /usr/bin/phpize /usr/share/man/man1/phpize.1.gz

[root@localhost mcrypt]# /usr/local/php/bin/phpize 
Configuring for:
PHP Api Version:         20090626
Zend Module Api No:      20090626
Zend Extension Api No:   220090626

执行完后，会发现当前目录下多了一些configure文件，最后执行php-config命令就基本完成了

执行以下命令，确保你的/usr/bin/php-config是存在的（如果不存在去【/usr/local/php/bin/php-config】）

[root@localhost mcrypt]# whereis php-config
php-config: /usr/bin/php-config /usr/share/man/man1/php-config.1.gz
[root@localhost mcrypt]# ./configure --with-php-config=/usr/bin/php-config(如果不存在填【/usr/local/php/bin/php-config】)
或者
[root@localhost mcrypt]# ./configure --with-mcrypt=/usr/local/mcrypt --with-php-config=/usr/bin/php-config(如果不存在填【/usr/local/php/bin/php-config】)

如果遇到以下错误，请先安装gcc，命令yum install gcc

configure: error: no acceptable C compiler found in $PATH

直到不报错，出现：config.status: creating config.h，执行以下命令

[root@localhost mcrypt]# make && make install

最后的最后，会提示你如下，说明你大功告成了

Installing shared extensions:     /usr/lib64/php/modules/（其它目录）

顺便检查下/usr/lib64/php/modules/里的mrcypt.so扩展是否已经创建成功

然后的事就简单了，给你的php.ini添加一条extension=mcrypt.so

[root@*_* 15:09 mcrypt]# cd /etc/php.d

创建一个mrcypt.ini文件就行，里面写extension=mcrypt.so

[root@*_* 15:17 php.d]# echo 'extension=mcrypt.so' > mcrypt.ini

以上两步修改为：

1、将生成的mcrypt.os复制到 /usr/lib64/php/modules（yum安装php时的扩展目录）

2、去到/etc/php.d目录，新建一个配置文件mcrypt.ini。内容如下：

; Enable mcrypt extension module
extension=mcrypt.so

3、重启/etc/init.d/httpd restart

查看测试页面<?php phpinfo(); ?>

如果有mcrypt模块扩展就安装成功了！
